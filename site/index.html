Quadratic breadth

Matching should reward network breadth, not spam volume. Ten independent validators matter more than one person refreshing the same tweet ten times.

How the mechanism works

A participant installs the September Attention Camera and optionally adds a QUAI payout address.

The camera captures visible public attention receipts from X/Substack/network pages.

The server normalizes tweet IDs and checks whether each tweet has been seen before.

New tweet IDs queue immediate 1 QUAI manual bounties. Duplicate tweet IDs queue validation signals.

Funding model

The duplicate side is quadratic because validation value should rise with independent breadth:

first_seen_payout = 1 QUAI validation_breadth(tweet) = unique operators × unique surfaces × time separation future_match(tweet) ∝ (Σ sqrt(valid_receipts_by_independent_operator))² spam_discount = duplicate_same_operator_same_surface_same_window → near 0

What counts

Counts should favor communications expanding through the network:

✓ different operators ✓ different audiences/surfaces ✓ later sightings after propagation ✓ visible engagement deltas ✓ receipts tied to real public pages × synthetic smoke × private/doxxing material × same operator refresh spam × duplicate ID pretending to be a new tweet
